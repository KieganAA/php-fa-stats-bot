# bot-stats Chrome extension

Расширение к [fa-stats-bot](../). Помогает быстро отправить лендинги из AIO
в bot-stats как новую подписку (Telegram-пуш каждые N часов).

## Что умеет

1. **Row-level кнопки на app.aio.tech** — content script находит
   `<aio-data-lander-relation>` и вставляет «＋» в начало строки. Клик —
   ленд попадает в подборку (локальный батч). Если ленд уже в одной из
   твоих подписок — у «＋» появляется маленький янтарный кружок и тултип
   со списком подписок-владельцев.

2. **Плавающая панель** (drag'n'drop, низ-справа по умолчанию). Показывает
   текущую подборку + две формы: «Создать подписку» (имя + частота +
   Создать) и «Добавить в подписку» (выпадающий список существующих +
   Добавить). Всё работает без открытия popup'а. Закрыл `×` — больше не
   появится, пока не включишь в настройках расширения.

3. **AIO right-click menu** — правый клик на ленде → «📌 В подборку
   bot-stats». То же, что «＋», но не нужно целиться в кнопку.

4. **Popup picker** — иконка расширения. Полный CRUD над подписками
   (пауза, удаление, переименование), плюс поиск по id / стране / названию
   с autocomplete. Полезно когда нужно собрать подписку из лендов с
   разных страниц.

5. **Selection context menu** — выдели список id'ов в любом тексте
   (например `33169, 205215, 170476`) → правый клик → «Отправить
   выделенное в bot-stats» → ленды попадают в очередь popup'а.

## Установка (dev)

1. Открой `chrome://extensions`, включи «Режим разработчика».
2. «Загрузить распакованное расширение» → выбери эту папку (`extension/`).
3. В Telegram-чате с ботом запусти `/extension_token`. Скопируй токен
   и API URL из ответа.
4. Кликни иконку расширения → «Открыть настройки» → вставь URL + токен,
   жми «Сохранить». Внизу должно появиться «✓ Подключено как ...».
5. Открой `app.aio.tech` — справа снизу появится мини-панель.

## Авторизация

Per-user Bearer-токен с префиксом `bsx_`. В БД хранится только SHA-256
хэш. `rotate()` инвалидирует старый; `revoke` (`/extension_token revoke`)
полностью отзывает доступ. Last-used-at обновляется на каждом запросе —
видно, когда расширение было активно.

## API endpoints (на стороне сервера)

Все под префиксом `/api/ext/`, авторизация: `Authorization: Bearer bsx_…`.

| Method | Path                  | Что делает                                |
|--------|-----------------------|-------------------------------------------|
| GET    | /me                   | проверка токена + identity                |
| GET    | /groups               | мои подписки                              |
| POST   | /groups               | создать (primitives + interval + name)    |
| PATCH  | /groups/{id}          | пауза / переименовать / сменить интервал  |
| DELETE | /groups/{id}          | удалить                                   |
| GET    | /landings?q=          | autocomplete: id / страна / название      |
| POST   | /resolve              | bulk-проверить пачку id, вернуть partition|

## Безопасность

- Токен длиной ~288 бит энтропии. Префикс `bsx_` помогает обнаружить
  утечки grep'ом.
- В БД хранится только SHA-256 — расшифровать обратно невозможно.
- CORS: разрешён любой origin (защищает токен, не origin), preflight на
  OPTIONS возвращает 204.
- Все запросы идут через HTTPS (расширение требует `https://*` host
  permission).
- При rotate старый токен немедленно инвалидируется.
